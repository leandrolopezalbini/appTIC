4. OPTIMIZACIÓN DE SHEETS API (Performance)
   ─────────────────────────────────────────────────────────────────────────
   
   ❌ buscadorMaestro() - Lectura completa de 6 hojas cada vez
      Impacto: Si tienes 1000 filas × 6 hojas = hasta 6000 celdas/búsqueda
      Solución: Implementar CACHE_SHEETS (lee 1x cada 5 min)
      Ahorro: 80% de lectura
   
   ❌ aplicarFormatoInsumos() - 1000 llamadas API (una por fila)
      Impacto: Si 1000 filas × 100ms/llamada = 100+ segundos
      Solución: Batch setBackgrounds() (1 sola llamada)
      Ahorro: 99% de latencia
   
   ❌ registrarInstalacionTablero() - 5+ getHeaderIndex() innecesarios
      Impacto: Búsquedas repetidas O(n) en headers
      Solución: Cache índices al inicio: idx = { estado: X, obs: Y }
      Ahorro: 70% de búsquedas
   
   ACCIÓN RECOMENDADA:
   ☐ Implementa precargarCacheHojas() (ver CORRECCIONES_CODIGO.gs)
   ☐ Convierte búsquedas lineales a Map/índices
   ☐ Batch writes con setBackgrounds()
   ⏱️ Tiempo: 3 horas
   
   IMPACTO: Sistema 5-10x más rápido
