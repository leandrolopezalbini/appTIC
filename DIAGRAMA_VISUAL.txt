                       FLUJO DE DATOS - ANTES
═══════════════════════════════════════════════════════════════════════════════

Frontend envía solicitud:
{tipoAccion: "ACTUALIZAR_ENERGIA", idTablero: "T-001"}
         │
         ▼
procesarSolicitudUnificada(data)
         │
         ├─→ if ((data.id || data.idTarea) && data.tipoAccion) {  ⚠ ASUME válido
         │        ... switch case ...
         │
         ├─→ notificarEvento()
         │        ├─ try {
         │        │    MailApp.sendEmail(...)
         │        │ }
         │        └─ catch(e) { }  ⚠ SILENCIO - Frontend nunca sabe
         │
         └─→ return { success: true }  ⚠ Siempre true, aunque haya fallado email


PROBLEMA ESPECÍFICO:
  1. Si falta data.tipoAccion → CRASH SILENCIOSO
  2. Si email falla → Frontend no lo sabe → Usuario no notificado
  3. Registro en Historial → pode FALTA si olvidas agregarlo
  4. Si búsquedas lentas → Usuario espera 5+ segundos
